# General Task Workflow

Use for questions, file inspection, explanations, comparisons, and lightweight
reviews.

## Steps

1. Read only the files needed to answer the question.
2. Do not create notes, reports, or screenshots unless the task becomes a meaningful implementation change.
3. If the request turns into a code or documentation change, switch to the matching workflow and read `docs/states/` first.
4. Answer with concrete file paths, command output summaries, or findings.

## Verification

Use direct evidence: file reads, command output, or targeted searches.
