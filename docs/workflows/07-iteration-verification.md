# Iteration And Verification Workflow

Use for long-running work, repeated failures, recovery, and unclear behavior.

## Steps

1. Restate the current task and known failure.
2. Read the state docs and the most relevant recent note or handoff.
3. Reduce the problem to the smallest reproducible check.
4. Change one variable at a time.
5. Keep a concise evidence trail in `notes/` when the work becomes meaningful.
6. Stop after the bounded task is resolved; do not auto-start the next task.

## Compression Rule

Prefer concise task context over repeatedly reading broad histories. Search
`notes/` for targeted keywords when prior BrandGen context is needed.
