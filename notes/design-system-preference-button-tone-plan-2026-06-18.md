# Design System Preference Button Tone Plan

Date: 2026-06-18

## Request

The header preference buttons look like they are wrapped multiple times, and the active background color is too loud.

## Before Screenshot

- `notes/screenshots/design-system-preference-button-tone-2026-06-18/before-fullscreen.png`

## Plan

1. Keep the existing single-button toggle structure for language and theme.
2. Remove the strong active color inversion that makes the active button visually pop too much.
3. Remove inset shadow/ring treatment that makes the icon button feel double-wrapped.
4. Keep fixed control widths so toggling does not shift header layout.
5. Verify the `/design-system` route and capture an after screenshot.
